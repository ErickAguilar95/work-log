# Work Log

Este repositorio funciona como una **línea de tiempo pública** de mi trabajo. Los commits se suben en modo **mirror desde GitLab** para mantener un historial unificado y cronológico de lo que voy desarrollando.

## Objetivo

- Tener un registro centralizado y ordenado de cambios día a día.
- Facilitar la consulta de qué se hizo y cuándo.
- Servir como respaldo y referencia del avance de proyectos.

## Estructura

```
changelog/{Month}-{Year}.md
```

```
# Changelog — June 2025
Todas las modificaciones relevantes de este mes se documentan aquí.

## [dd-mm-YYYY] - titulo
descripcion breve de lo que se hizo, por ejemplo:
- Se implementó la nueva funcionalidad X en el proyecto Y.
- Se corrigió el bug Z que afectaba a los usuarios en la versión W.
- Se mejoró el rendimiento del módulo A en un 20%.
```

Los cambios son solo informativos, por politicas de seguridad de trabajos actuales
